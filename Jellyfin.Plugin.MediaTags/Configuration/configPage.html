Media Tags

Help

This plugin scans your movies, tv shows and collections for the properties of their video streams and adds tags to the items according to their resolutions.

Additionally the plugin can color range tags for HDR and Dolby Vision, if enabled.

Whitelist media Tags (Optional!)

Leave empty to allow all tags, or select specific tags below.

Only selected tags will be added to your media. If no tags are selected, all detected tags will be added.

Custom Tag Prefixes

Customize the prefix for resolution tags. Default is "res_". Tags will be created as "[prefix][resolution_name]", e.g., "res_1080p", "res_sdr".

Leave empty to use default. If specified, must be at least 3 characters and different from color range prefix.

Customize the prefix for color range tags. Default is "range_". Tags will be created as "[prefix][hdr_type]", e.g., "range_HDR", "range_DV".

Leave empty to use default. If specified, must be at least 3 characters and different from resolution prefix.

HDR Tags

When enabled, the plugin will add color range tags e.g. "range_HDR" based on the available metadata.

TV Show Tagging

When enabled, media tags are applied only to the root Series item. Episode and season media data is still read and aggregated upward — only the tagging of seasons and episodes is skipped. This prevents Jellyfin's tag view from being flooded with hundreds of individual episode entries.

Note: Enabling this does not remove previously applied tags from episodes/seasons. Use Remove ALL media tags first if you want a clean state.

Full Refresh Options

When enabled, the plugin will always perform a full refresh of the library when scanning for media tags. This is resource intensive and should only be enabled if you are experiencing issues with the plugin.

Non-Media Item Tagging (Optional)

When enabled, applies a tag to non-media items (music, books, people, etc.) so they appear when using "Allow items with tags" in Parental Control.

Use Case: If you use "Allow items with tags" to filter content by language, non-media items (artists, albums, people, etc.) will be hidden. This feature tags those items so they remain visible.

The tag name to apply to non-media items. Default is "item". Make sure to add this tag to your "Allow items with tags" list in Parental Control settings.

Select Item Types to Tag:

Choose which types of content should receive the non-media tag. Only selected types will be tagged.

Synchronous Refresh Option

NOT RECOMMENDED

When enabled, the plugin will perform a synchronous refresh of the library when scanning for media tags. This takes significantly longer, but is less ressource intensive.

Usefull for low powered setup like a Raspberry Pi.
